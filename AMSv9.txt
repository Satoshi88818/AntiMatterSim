AntimatterSim v9 — Level 9 Full Codebase

Upgrade from v8 (Level 8) to v9 (Level 9): Autonomous Molecular Precision

This release transitions AntimatterSim from a Clinical Research Platform to an Autonomous Adaptive Treatment System. The five pillars of Level 9 are:

4D Deformable Geometry — Time-resolved CT (4DCT) with breathing/cardiac motion via Deformable Image Registration (DIR)

Inverse Planning (IMAT) — Goal-based beam optimization: given a desired dose distribution, compute the beam configuration

Molecular DNA Damage Modeling — Micro-Kinetic Models (MKM) and Double-Strand Break (DSB) prediction at nanometer scale

Physics-Informed Neural Networks (PINNs) — Bethe-Bloch and energy-conservation constraints embedded in surrogate loss; Conditional GAN for PET-to-Dose inversion

Closed-Loop Beam Adaptation — Millisecond gRPC streaming for real-time beam steering; Kubernetes orchestration for population-scale trials

""" deformable_dir.py (v9) ====================== Deformable Image Registration (DIR) for 4D patient geometry. Level 9 upgrade --------------- Level 8 treated the patient as a static 3D CT volume. Level 9 models the fact that tumours move — a lung tumour can displace by 15–30 mm during breathing, and cardiac motion affects thoracic structures. Failing to account for this leads to geographic miss (beam misses the tumour) or excessive dose to organs at risk. Algorithm --------- This module implements two DIR approaches: 1. **B-spline Deformation (default)**: - Control points placed on a regular grid (spacing ~10 mm). - Each voxel position p is displaced by: p' = p + Σ_{i} c_i * B_i(p) where c_i are control point displacements and B_i are cubic B-spline basis functions. - GPU-accelerated via bilinear/trilinear texture interpolation (simulated here via scipy.ndimage.map_coordinates with order=3). 2. **Optical Flow (Lucas-Kanade variant)**: - Computes a dense velocity field v(x,y,z) from intensity gradients between consecutive 4DCT phases. - Each voxel is advected: p'(t) = p + v(p) * Δt - Suitable for large-deformation cardiac motion. The DeformableGeometry class wraps a list of DICOMGeometry phases and provides a unified material_at_position_cm(x, y, z, t) interface where t is simulation time. The engine calls this at each transport step to look up the current material, automatically accounting for motion. References ---------- Rueckert et al., IEEE Trans. Med. Imaging 18 (1999) 712 — B-spline DIR. Horn & Schunck, Artif. Intell. 17 (1981) 185 — optical flow. Keall et al., Med. Phys. 33 (2006) — 4DCT for radiation therapy. """ from __future__ import annotations import numpy as np from dataclasses import dataclass, field from typing import List, Optional, Tuple try: from scipy.ndimage import map_coordinates _HAS_SCIPY = True except ImportError: _HAS_SCIPY = False @dataclass class BSplineField: """ B-spline deformation vector field for one phase transition. Attributes ---------- control_points : np.ndarray, shape (Cx, Cy, Cz, 3) Displacement vectors at each control point [cm]. grid_spacing_cm : float Control point spacing [cm]. volume_shape : tuple of 3 ints Shape (nx, ny, nz) of the target volume. origin_cm : np.ndarray, shape (3,) World-space origin of the volume [cm]. """ control_points: np.ndarray # (Cx, Cy, Cz, 3) displacement vectors [cm] grid_spacing_cm: float # control point spacing volume_shape: Tuple[int, int, int] origin_cm: np.ndarray # (3,) world origin @dataclass class DeformationPhase: """One phase of a 4DCT series with its deformation field from the reference.""" phase_index: int geometry: object # DICOMGeometry for this phase deformation_field: Optional[np.ndarray] = None # shape (nx,ny,nz,3) [cm] or None for reference class BSplineDeformableRegistration: """ B-spline Deformable Image Registration. Computes the displacement vector field (DVF) that maps voxels from a reference phase to any other phase. Parameters ---------- reference_phase : DICOMGeometry The reference CT phase (usually end-exhale, phase 0). grid_spacing_mm : float B-spline control point spacing [mm]. rng : np.random.Generator For stochastic initialisation in optimisation (test/demo mode). """ def __init__( self, reference_phase, grid_spacing_mm: float = 10.0, rng: Optional[np.random.Generator] = None, seed: int = 0, ): self._ref = reference_phase self._grid_spacing_cm = grid_spacing_mm / 10.0 self._rng = rng or np.random.default_rng(seed) def register( self, moving_phase, max_iterations: int = 100, ) -> np.ndarray: """ Compute the DVF that maps moving_phase → reference_phase. In production, this calls ANTs (antspyx) for clinical-grade registration. This implementation uses a simplified gradient-descent intensity-based registration for testing and demonstration.
